What is Waltz?

Waltz is the open source platform that gives your organisation a living, connected map of the technology landscape: applications, data flows, people, capabilities and change initiatives in one place.

It helps leaders, architects, engineering teams and risk functions align decisions with evidence.

Why Architecture Decisions Stall

Scattered & Stale

Architecture data lives in spreadsheets, disconnected tools and people's heads. By the time it's consolidated, it's already out of date.

Hard to Navigate

Finding how a business capability maps to applications, data flows and org units requires jumping between a dozen systems.

Compliance Risk

Regulatory programs like DORA, MiFID II and Basel require accurate application topology — but most firms can't produce it on demand.

Features

See your applications

Waltz presents data about each application and how it connects to capabilities, organisational units, people, servers, data types and more. Once linked, relationships become navigable and decision ready.

Information can be augmented manually to increase the completeness and accuracy of application views used by architects, delivery teams and auditors.

Aggregate Views

Once applications have been linked to entities (such as organisational units, people and capabilities) you can navigate through dynamic views which show aggregate summaries of related applications, costs, ownership and criticality.

See how an application is connected

How an application is connected to other applications in your organisation is a central part of Waltz. We can describe upstream and downstream applications along with the data types exchanged.

As this topology is created you can define authoritative sources for data types and see which applications source data from approved golden sources, supporting data governance and regulatory reporting.

See how your organisation is connected

Building upon an individual application's connections we can start to view how applications are connected when they are aggregated.

If authoritative sources have been defined an organisational-unit level scorecard shows what proportion of logical data flows are sourced from strategic systems, helping prioritize remediation and investment.

Statistics

Define your own sets of statistics and use Waltz to navigate collected values by the structural entities relevant to you, supporting portfolio health, risk and compliance reporting.

Create and share custom groups

Waltz allows you to create custom, ad-hoc groups of applications. This feature is often used to quickly understand a subset of your application portfolio. For instance you may wish to create a group containing SOX relevant applications or systems in scope for DORA, GDPR or PCI DSS.

How Waltz fits into the EA Tooling landscape

Waltz is designed to complement existing architecture-related solutions in your organisation (such as PowerDesigner, Archimate, Sparx or other Enterprise Architecture tools); it complements these solutions and enables integration with them as the need arises. Waltz is not intended to replace modeling tools or become a single golden source beyond attributes derived in the tool (e.g. estate complexity scores).

As a general principle, Waltz presents read-only information about the organisation by aggregating data from numerous backend systems. Where information is not readily available Waltz provides basic edit capabilities to facilitate data capture. This supports evidence-based responses for regulatory and audit programs such as DORA, BCBS 239, SOX, GDPR and PCI DSS by making scope, ownership, and flow evidence easier to assemble and review.

Technical Details

Client Side

Waltz is accessed via a web browser. Modern web browsers are supported. We have explicitly tested on:

  • Chrome
  • Firefox
  • Safari
  • IE 11+

The application makes heavy use of AngularJS, lodash and D3 for interaction. Bootstrap and Fontawesome for look n' feel and is built using Webpack and SASS.

Security

Waltz has a pluggable security system and can authenticate users against a built in set of user profiles or integrate with external systems such as SSO. Contact us for more details.

Once authenticated, users can be granted discrete privileges within the application for authorization purposes. New users can be granted a default set of permissions; we recommend a generous set of defaults as Waltz works best when a diverse set of users contribute their knowledge.

Server Side

Waltz is built as a Java based web application. We currently deploy to (embedded) Jetty and to Apache Tomcat (10+) server. JDK 17 is required to build and run Waltz.

We recommend a 64 bit JVM running with a minimum of 1GB heap.

Under the covers Waltz uses Spark Framework to expose endpoints, Spring for dependency injection, JOOQ for database access and we use Immutables for representing the domain object layer.

Database

Waltz is being built to support PostgreSQL (currently supported), and Microsoft SQL Server (supported — however you will need to purchase a license for JOOQ to compile and build the system).
