Cloud migration programmes usually start with one question: which applications are ready to move?

It rarely has a simple answer.

<!--more-->

An application can be technically pristine but regulatorily blocked. Business-critical but operationally fragile. Cloud-ready but close to retirement. A single "ready/not ready" score hides exactly the distinctions you need to make a migration plan you can defend.

Our latest post sets out a practical readiness model built in Waltz, scoring applications across six dimensions: **technical**, **operational**, **business**, **lifecycle**, **dependency**, and **regulatory**. Mapping the result to an eight-value rating scheme that points directly to a treatment: rehost, replatform, refactor, replace, retain, or retire.

The aim is not another framework. It is a model that helps you:

- Separate strong migration candidates from blockers
- Surface low-ROI applications
- Sequence migration waves on evidence rather than opinion
